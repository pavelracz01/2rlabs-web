# Transformace QA v startupu

15-členný SaaS startup přišel s kritickým problémem: každý release něco rozbil v produkci.

## Výzva

- Žádný formální QA proces
- 2 vývojáři dělali manuální smoke testing před release
- Chyby hlášené zákazníky přibývaly každý měsíc

## Co jsme udělali

1. **Týden 1-2:** Audit kódu a release procesu
2. **Týden 3-4:** Definice testovací strategie a setup Playwright
3. **Měsíc 2:** 80% pokrytí kritických cest
4. **Měsíc 3:** CI/CD integrace a školení týmu

## Výsledky

- 70% snížení produkčních chyb
- Release cyklus ze 2 týdnů na týdenní
- Výrazně vyšší sebedůvěra týmu při vydávání funkcí
